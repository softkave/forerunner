# `softkave-forerunner` `v0.x.x` - Release Notes

## v0.15.3

- Fixed file permission issue when starting MongoDB instances.

## v0.15.0

- Moved to Docker for running containerized MongoDB instances.
- Enhanced `instancesHostnames` in `MongoRunConfig` to support object format with `hostname` and `resolution` (`"dns"` or `"local"`) fields, allowing control over Docker bridge IP binding. Arrays can now mix strings and objects. Backwards compatible with string/string[] formats.
- Removed `replicaCount` from `MongoRunConfig`; instance count now derived from `instancesHostnames.length` (must match `instancePorts.length`, minimum 3).
- Made `replicaSetName` required in `MongoRunConfig`.
- Enhanced `setupUsers` to synchronize users with config: adds new users, updates roles/passwords for existing users, removes users not in config (with safeguard to prevent removing sole admin).
- Added `mongo restart` command for rolling restart of replica set members (useful for version upgrades and config changes).
- Improved instance startup: checks if Docker containers are already running before attempting to start.

## v0.14.0

- Added `run-env` sub-command: discovers `.env*` files in the current directory, lets you pick one (interactive list if multiple), then runs a command after `--` with that env loaded and I/O piped through. Options: `--cwd`, `--silent`.

## v0.13.0

- Daily rotate MongoDB log files.
- Changed default MongoDB Community version to `8.2.3`.

## v0.12.0

- Enhanced `startProcess` function to create processes with their own process group IDs (PGID) for better cleanup management.
- Added `stopProcessGroup` option to `endPIDs` function to terminate entire process groups using PGID.
- Added PGID support to PID file format and process management functions.
- Enhanced dummy server with stderr logging endpoint for comprehensive testing.

## v0.11.0

- Enhanced `endPIDs` function with `stopChildren` option to optionally terminate child processes.
- Renamed `killAfterMs` parameter to `timeoutMs` for better clarity.

## v0.10.0

- Added `pm children-pids` command to find all child process IDs of a given parent PID recursively.
- Added `findChildrenPIDs` SDK function for programmatic access to child process discovery.
- Updated dependencies: commander to `v14.0.1`.
- Enhanced dummy server to return both Express app and HTTP server instances.

## v0.9.3

- Updated the `endPIDs` function to use Node.js' `process.kill(...)`.

## v0.9.2

- Use cluster admin for `mongo replica-set-status` and `replicaSetStatus`.

## v0.9.1

- Added the ability to choose where to get replica set status when running `mongo replica-set-status` using `--ping`. It provides `all`, `repl`, and instance number. See README for more information. This is also available using the SDK function `replicaSetStatus` which `mongo replica-set-status` uses underneath.

## v0.9.0

- Added `mongo replica-set-status` command for printing a running MongoDB replica set's status, using supplied config.
- Updated the format for printing `etc-hosts list` from table to list.

## v0.8.0

- Added `mongo print-uri` command to print MongoDB connection URIs using configuration.
- Added `printMongoUriMain` function to generate and display MongoDB URIs for both instance and replica set connections.
- Support for authentication credentials, localhost preference, and configurable server selection timeout in URI generation.

## v0.7.4

- Bumped default downloaded MongoDB Community version to `8.0.13`, the latest version as of this change.
- Added `readAnyDatabase` preset role and allow adding any role in `MongoUser`.
- Added the `readAnyDatabase` role to auto-generated admin users.
- Disabled the use of symbols yet again in generated passwords.
- Check for instance ready state (instances are open for new connections & replica set is ready) both on `mongo init` and `mongo start` commands. Also updated how the checks work. The checks use system logs and should now be able to handle log files of any size.
- Fixed an issue where ready state checks used the first instance's logs when instances restart.
- Fixed an issue where downloading certain Mongodb versions fail when extracting and moving binaries.
- Fixed an issue where `setupSingleMongoUser` throws an error when adding a user that already exists, rather than logging that the user exists and gracefully return.
- Made the following changes to `startMongodInstancesMain`:
  - `waitUntilListening`, so the call waits until instances are ready to accept connections. It times out after 15 seconds.
  - `waitUntilReplicaSetReady`, the call waits until the replica set is ready. It times out after 15 seconds.

## v0.7.3

- Exclude the following symbols `"'\` (back slash, single quote, and double quote) when generating passwords, for convenience when working with password in code and config.

## v0.7.2

- Fixed an issue where a new admin and cluster admin was created, different from what's in `mongoRunConfig` and `mongo-users.json` causing authentication issues.
- Re-enabled using symbols in generated passwords used for `mongo` setup.

## v0.7.1

- Disabled using symbols in generated passwords used for `mongo` setup.

## v0.7.0

- Fixed an issue where admin and cluster admin users from `mongoRunConfig`/`--config` or `usersFilePath`/`--users` weren't found, because the function was looking only at the existing `mongo-users.json` when writing users through `mongo write-users` or `writeMongoUsersFromConfig`.

## v0.6.7

### CLI

- Improved documentation of CLI commands and configurations.
- Renamed `mongo run` to `mongo init`, seeing it's meant to be run once and subsequent operations should be carried out using `start` and `stop`. Also removed `--addToEtcHosts` option from `mongo init`, because it's better run separately using `mongo etc-hosts`.
- Changed `--users` filepath option from required to optional when running `mongo write-users`, and added `--create-admin` (to create add an admin user if not present) and `--create-cluster-admin` (to add a cluster admin user if not present) options.
- Updated `mongo etc-hosts` command to map non-IP instance hostnames in `mongoRunConfig` to the system's local IPv4 address if present but default to `127.0.0.1` if there's no IPv4 address.

### SDK

- Made the following changes to `MongoConfig` when generating Mongodb config yaml files through `generateMongoConfigForMongod`:
  - Allow `enabled` and `disabled` options for `security.authorization`. Previously, only the `enabled` option was available.
  - Added `security.transitionToAuth` to disable authentication (meant to be used temporarily), which is useful when adding users to a newly setup replica set that contains non-localhost members (basically any hostname that's not `127.0.0.1` and `localhost`).
- Allow modifying `MongoConfig` using `modifyConfig` function parameter when generating Mongodb config yaml files through `generateMongoConfigForMongod`.
- Updated `setNonLocalhostNamesInEtcHosts` and consequently `setNonLocalhostNamesInEtcHostsMain` to map non-IP instance hostnames in `mongoRunConfig` to the system's local IPv4 address if present but default to `127.0.0.1` if there's no IPv4 address.
- Made the following changes to `setupSingleMongoUser`:
  - Added `client` (for reusing mongodb clients),
  - Added `retainClient` (return mongodb client and do not close it after setup), and
  - Added `connectionType` (`'replicaSet' | 'instance'`).
- Renamed `setupReplicaSetFirstUsers` to `setupFirstUsers` and made the following changes:
  - Added `client` (for reusing mongodb clients), and
  - Added `connectionType` (`'replicaSet' | 'instance'`).
- Made the following changes to `setupReplicaSetMain`:
  - Added `retainClient` (return mongodb client and do not close it after setup).
- Added `writeMongoUsersFromConfig` (a wrapper around `writeMongoUser`) which provides merging users from `mongoRunConfig` with `usersFilePath`, and provides other options like whether to create admin (`createAdmin`) and cluster admin users (`createClusterAdmin`) if not present.

## v0.6.6

- Fixed an issue where the first character of an hostname was used when compiling a uri for a replica set, when `instanceHostnames[index]` is a string.

## v0.6.5

- Fixed an issue where filtering for local hostnames returned non-local hostnames.
- Prefer using non-local hostnames when setting up a replica set (as per mongodb docs recommendation), but choose the first `bindIp` otherwise.

## v0.6.4

- Prefer using a local instance (connect to an instance bound to `localhost`/`127.0.0.1`) when setting up a replica set or first users (when running `mongo run ...`), to take advantage of local authentication bypass when `security.authorization` is `enabled`.

## v0.6.3

- Use system logs to check if mongo instances have started and when the replica set is ready, rather than waiting an arbitrary amount of minutes.
